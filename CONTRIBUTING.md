# Contributing to the Ethos AXS Suite

Thank you for your interest in contributing to this open innovation proposal.

## Areas of Contribution

**Physics Model (Payload Oracle)**
- Improved energy consumption models accounting for tyre compound, temperature
- Validation against real-world EV telemetry datasets
- Additional vehicle platform specifications

**Acoustic DSP (ARE)**
- Additional Heritage acoustic profiles (e.g., inline-6, V10 supercar)
- Improved RPM-equivalent mapping algorithms
- AVAS compliance validation tools

**Driver Profiler (AGA)**
- ML model architecture proposals (federated learning)
- Additional blind-spot detection heuristics
- GDPR-compliant on-device training frameworks

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/improved-physics-model`)
3. Add tests for any new functionality
4. Ensure all existing tests pass (`python -m pytest tests/`)
5. Submit a pull request with a clear description of the changes

## Code Standards

- Python 3.10+
- Type hints on all public functions
- Docstrings following Google style
- Unit tests for all calculation functions

## Contact

For major proposals or partnership enquiries, open a GitHub Issue with the label `innovation-proposal`.
